## Chomsky Hierarchy

The **Chomsky Hierarchy**, proposed by Noam Chomsky, is a fundamental classification of formal languages based on the **restrictions of their grammars** and the **computational power required to recognize them**. It is central to Theory of Computation and compiler design.

The hierarchy consists of four levels:

* Type-3: Regular Languages
* Type-2: Context-Free Languages
* Type-1: Context-Sensitive Languages
* Type-0: Recursively Enumerable Languages

### Definition

Regular languages are the simplest class of languages, defined by **regular grammars** and recognized by **finite automata**.

### Grammar Rules (Regular Grammar)

Two forms:

* Right-linear:
  A → aB | a
* Left-linear:
  A → Ba | a

### Computational Model

* Recognized by:

  * Deterministic Finite Automaton (DFA)
  * Non-deterministic Finite Automaton (NFA)

### Example

Language:
L = { strings over {0,1} ending with "01" }

Regular Expression:

```
(0|1)*01
```

### Key Characteristics

* No memory beyond current state
* Cannot handle nested or recursive patterns
* Highly efficient

### Applications

* Lexical analysis in compilers
* Pattern matching (e.g., regex engines)

### Definition

Languages generated by **context-free grammars (CFGs)** where each production rule has a single non-terminal on the left-hand side.

### Grammar Rules

```
A → γ
```

Where:

* A = non-terminal
* γ = string of terminals and/or non-terminals

### Computational Model

* Recognized by **Pushdown Automata (PDA)**
* Uses a **stack** for memory

### Example

Language:
L = { aⁿbⁿ | n ≥ 0 }

Grammar:

```
S → aSb | ε
```

### Key Characteristics

* Can represent nested structures
* Limited memory via stack
* Cannot handle multiple dependencies (e.g., aⁿbⁿcⁿ)

### Applications

* Syntax analysis (parsing)
* Programming language design

### Definition

Languages generated by **context-sensitive grammars**, where productions depend on the surrounding symbols (context).

### Grammar Rules

```
αAβ → αγβ
```

Constraint:

* Length of output ≥ length of input (no shrinking)

### Computational Model

* Recognized by **Linear Bounded Automata (LBA)**
* Tape size is limited to input length

### Example

Language:
L = { aⁿbⁿcⁿ | n ≥ 1 }

This language requires:

* Equal number of a, b, and c
* Not possible with CFG

### Key Characteristics

* More powerful than CFL
* Can enforce multiple dependencies
* Requires bounded but non-trivial memory

### Definition

The most general class of languages, generated by **unrestricted grammars** and recognized by a Turing Machine.

### Grammar Rules

```
α → β
```

(No restriction, except α is non-empty)

### Computational Model

* Turing Machine (infinite tape, read/write head)

### Example

Language:
L = { w#w | w ∈ {0,1}* }

### Key Characteristics

* Maximum computational power
* May not halt for some inputs
* Includes undecidable problems

### Applications

* General computation
* Formalizing algorithms

## Relationships Among Language Classes

### Inclusion Property

The hierarchy is strictly nested:

```
Regular ⊂ Context-Free ⊂ Context-Sensitive ⊂ Recursively Enumerable
```

### Interpretation

* Every **regular language** is also **context-free**
* Every **context-free language** is also **context-sensitive**
* Every **context-sensitive language** is **recursively enumerable**
* The reverse inclusion does not hold

## Comparative Summary

| Type | Language Class         | Grammar Restriction       | Machine Model            | Memory Type  |
| ---- | ---------------------- | ------------------------- | ------------------------ | ------------ |
| 0    | Recursively Enumerable | No restriction            | Turing Machine           | Infinite     |
| 1    | Context-Sensitive      | Length non-decreasing     | Linear Bounded Automaton | Bounded tape |
| 2    | Context-Free           | Single non-terminal (LHS) | Pushdown Automaton       | Stack        |
| 3    | Regular                | Linear rules              | Finite Automaton         | No memory    |

---

## Conceptual Understanding

* **Regular Languages** → Simple patterns, no memory
* **Context-Free Languages** → Nested structures (e.g., parentheses)
* **Context-Sensitive Languages** → Multiple dependencies
* **Recursively Enumerable Languages** → General computation
